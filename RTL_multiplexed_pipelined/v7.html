RTL Neural Equalizer Design Atlas

Detailed design documentation for the current time-multiplexed pipeline in neural_eq_top.sv, including dataflow blocks, control timing, ROM packing, and script-driven verification.

0. Executive Summary

The design implements a compact 3-layer neural equalizer using shared Triple-MAC compute lanes, staged valid handshakes, and fixed-point Q1.14 arithmetic. Throughput is symbol-paced by the testbench at one symbol every 6 clocks.

Input: Signed 16-bit in_I and in_Q, valid-gated by valid_in.
Window: 5 complex taps (10 real scalar values).
L1: 10 -> 32, 4 ticks.
L2: 32 -> 32, split A/B halves, 6 ticks.
L3: 32 -> 2, split A/B halves, 6 ticks.
Output: Saturated 16-bit out_I,out_Q with pulse valid_out.

2. Smart Input Window and Flush Logic

The front-end waits until enough symbols are present, then emits window-valid pulses. During long silence, it automatically pushes INIT_VAL into the window to drain residual pipeline state.

Signal Role Behavior
fill_cnt Window warm-up Counts initial valid symbols; first normal v_win occurs when third symbol arrives.
silence_timer Idle detect Increments only after warm-up, when no valid input and not already flushing.
flushing_active Flush mode gate Starts after timeout and injects deterministic samples to complete pending computations.
flush_cnt Flush phase index Drives two window pushes (at count 0 and 6) then exits flush mode.
if (valid_in) silence_timer <= 0; else if (fill_cnt >= 3 && !flushing_active) silence_timer++; ... if (flush_cnt == 0 || flush_cnt == 6) v_win <= 1;
Why this matters: file-level output alignment becomes cleaner because tail symbols are intentionally flushed instead of being silently dropped.
3. Per-Layer Compute Diagram (Block-Like View)

Each layer uses an explicit FSM style: idle -> busy/tick sweep -> fire valid.

Layer 1 (10 -> 32, 4 ticks)

  • Latch 5 I and 5 Q values when v_win pulses.
  • Tick 1..4 select 3 scalar inputs per cycle via MUX case logic.
  • Per-neuron accumulator computes weighted sum then applies bias and ReLU.
  • Fires v_l1 once at tick 4 completion.

Layer 2 (32 -> 32, 6 ticks)

  • Input vector split into A and B halves (16 + 16 scalars).
  • A and B halves are processed in parallel each tick (6 products total per neuron per tick).
  • No latency doubling from split; still 6 ticks end-to-end.
  • Re-latch support allows back-to-back v_l1 handoff without dead cycle.

Layer 3 (32 -> 2, 6 ticks)

  • Same split-A/B template as Layer 2.
  • Computes two output neurons: I and Q.
  • Final stage is saturation-only (no ReLU).
  • Asserts valid_out on tick 6 completion.
Layer Input Width Ticks Activation Output Valid Pulse
L1 10 scalars (I/Q window) 4 ReLU + Saturation v_l1
L2 32 scalars (split 16+16) 6 ReLU + Saturation v_l2
L3 32 scalars (split 16+16) 6 Saturation only valid_out

5. ROM Packing Diagram (Triple-Width 48-bit)

6. End-to-End Tool Flow (Prep -> Sim -> Verify)

The Makefile and Python scripts form a reproducible loop from model dumps to constellation quality metrics.

Step 1: HEX Generation

scripts/01_prep_hex.py loads txt arrays, quantizes to 16-bit hex, packs triple-weights, and writes stimulus stream.

Step 2: Compile

iverilog -g2012 compiles tb/tb_neural_eq_top.sv with all RTL sources.

Step 3: Run

vvp sim_data/sim.vvp executes simulation and writes sim_data/rtl_output.txt.

Step 4: Verify

scripts/02_verify.py auto-aligns delay, computes MSE, SER, BER, and plots constellation overlays.

all: prep compile sim verify wave
7. Practical Debug Checklist
Mismatch at startup: Confirm warm-up behavior and first valid_out index.
Wrong amplitudes: Re-check Q1.14 scaling and VOLT_MULTIPLIER in verification.
Large MSE: Inspect weight ordering in packed 48-bit rows.
Missing tail symbols: Verify flush timeout and flush injection pulses.
Output clipping: Check saturation boundaries +32767/-32768.
Timing confusion: Trace pulse chain v_win -> v_l1 -> v_l2 -> valid_out.
