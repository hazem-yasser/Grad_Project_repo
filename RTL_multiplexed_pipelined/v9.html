Signal Dictionary

Signal-by-signal explanation for rtl/neural_eq_top.sv plus key testbench signals.

How To Read This

Each row maps one signal name to its purpose, when it changes, and why it exists.

INPUT external input to module.
OUTPUT final output from module.
CTRL handshake/FSM/timing signal.
DATA arithmetic/data-path signal.
MEM ROM/storage signal.
1) Top-Level Ports
Signal Type Meaning When active/updated
clkINPUT CTRLMain system clock.All sequential logic updates on posedge.
rst_nINPUT CTRLActive-low reset for all layer state, accumulators, and outputs.Whenever low, most state is cleared.
valid_inINPUT CTRLMarks incoming sample pair in_I/in_Q as valid.Pulsed by TB once every 6 cycles.
in_IINPUT DATAInput in-phase sample, signed 16-bit.Consumed when valid_in=1.
in_QINPUT DATAInput quadrature sample, signed 16-bit.Consumed when valid_in=1.
out_IOUTPUT DATAFinal equalized I output after L3 saturation.Updated on L3 tick 6 completion.
out_QOUTPUT DATAFinal equalized Q output after L3 saturation.Updated on L3 tick 6 completion.
valid_outOUTPUT CTRLOne-cycle pulse indicating out_I/out_Q are valid.Asserted when L3 reaches final tick.
2) Smart Window Signals
Signal Type What it does Notes
INIT_VALDATAConstant fill value used on reset and flush injection.Signed 16-bit parameter.
TIMEOUT_CYCLESCTRLSilence duration before auto-flush starts.Parameter set to 50.
win_I[0:4]DATAShift-register window of recent I samples.Moves left each valid input or flush event.
win_Q[0:4]DATAShift-register window of recent Q samples.Paired with win_I.
v_winCTRLPulse telling L1 "window is ready this cycle".Generated after warm-up and during flush pushes.
fill_cntCTRLTracks early valid symbols until window is primed.First true window pulse begins at 3rd symbol.
flush_cntCTRLStep index for flush sequence.Special pushes at counts 0 and 6.
silence_timerCTRLCounts idle cycles after warm-up.Resets on every valid_in.
flushing_activeCTRLEnables auto-flush state machine.Set when silence exceeds timeout.
3) Layer 1 Signals
Signal Type What it does When to watch it
l1_out[0:31]DATAL1 neuron outputs after bias+ReLU+saturation.Updated at tick 4 finalize cycle.
v_l1CTRLL1 done pulse toward L2.One cycle pulse at end of L1 run.
l1_latched_I[0:4]DATASnapshot of I window for stable compute.Captured when L1 starts.
l1_latched_Q[0:4]DATASnapshot of Q window for stable compute.Captured when L1 starts.
l1_tickCTRLCurrent sub-cycle (1..4) inside L1 MAC sweep.Main L1 timing signal.
l1_busyCTRLIndicates L1 engine currently computing.Blocks re-entry while active.
L1_W_ROM[0:127]MEMPacked L1 weights from hex file.Addressed by neuron index + tick.
L1_b_ROM[0:31]MEML1 bias per neuron.Added after accumulator scaling.
l1_in1,l1_in2,l1_in3DATASelected scalar triplet for current L1 tick.Driven by case(l1_tick).
Inside each L1_MAC generated neuron block
  • w_bus: 48-bit packed weights for this neuron and this tick.
  • p1,p2,p3: products for each selected input and weight lane.
  • acc: running sum across the 4 L1 ticks.
  • final_acc/tmp: finalize path before ReLU and saturation.
4) Layer 2 Signals
Signal Type What it does Key idea
l2_out[0:31]DATAL2 neuron outputs after bias+ReLU+saturation.Feeds L3 when v_l2 pulses.
v_l2CTRLL2 done pulse toward L3.One cycle at l2_tick==6 completion.
l2_tickCTRLSub-cycle index 1..6 for L2.Defines MUX and ROM slice.
l2_busyCTRLMarks L2 engine active.Supports re-latch when back-to-back.
l2_latched_in[0:31]DATAStable copy of L1 outputs for L2 pass.Captured on L2 start and optional re-latch.
L2A_W_ROMMEMWeights for first 16-input half.Parallel half A path.
L2B_W_ROMMEMWeights for second 16-input half.Parallel half B path.
L2_b_ROMMEMBias values for 32 L2 neurons.Added at finalization.
l2a_in1..3DATAThree selected inputs from half A.Chosen by case(l2_tick).
l2b_in1..3DATAThree selected inputs from half B.Chosen in parallel to half A.
Inside each L2_MAC generated neuron block
  • w_bus_a/w_bus_b: packed weights for A and B halves.
  • p1..p3: products from half A, p4..p6 from half B.
  • acc: running sum over 6 ticks (all six products each tick).
  • final_acc/tmp: finalize, scale, add bias, ReLU, saturate.
5) Layer 3 Signals
Signal Type What it does Difference vs L2
l3_tickCTRLSub-cycle index 1..6 for output layer.Same tick structure as L2.
l3_busyCTRLIndicates L3 processing active.Drives output validity timing.
l3_latched_in[0:31]DATAStable copy of L2 outputs for L3 pass.Captured when v_l2 starts job.
L3A_W_ROM/L3B_W_ROMMEMOutput-layer half-A/half-B weight tables.Only 2 neurons total.
L3_b_ROM[0:1]MEMBias for output I and Q neurons.Applied before saturation.
l3a_in1..3 / l3b_in1..3DATAPer-tick selected triples from each half.Same MUX pattern as L2.
Inside each L3_MAC generated neuron block
  • w_bus_a/w_bus_b: current packed weights for neuron and tick.
  • p1..p6: six multipliers (3 from each half).
  • acc: accumulated sum across ticks.
  • tmp: final scaled + biased result with saturation only (no ReLU).
  • if (i==0) out_I else out_Q: maps generated neuron index to I/Q outputs.
6) Key Testbench Signals (`tb/tb_neural_eq_top.sv`)
Signal Purpose Behavior in this TB

in_I/in_QDUT stimulus samples.Loaded from stimuli_mem[i*2], stimuli_mem[i*2+1].

7) Minimal Debug Order (Signal Trace Priority)

When debugging, inspect these in order to find where data stops:

valid_in and in_I/in_Q
v_win, fill_cnt, flushing_active
l1_busy/l1_tick -> v_l1
l2_busy/l2_tick -> v_l2
l3_busy/l3_tick -> valid_out
out_I/out_Q values on valid cycles
